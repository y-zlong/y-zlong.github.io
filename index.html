Posted on 2020-07-22T17:57:18 | In spring

spring是一个轻量级的企业开发框架。

1. IOC

ioc 容器对象，spring会帮我们管理对象，同时帮我们创建好对象，在spring中获取ioc容器对象时spring为我们提供了两个接口BeanFactory ApplicationContext可以获取ioc容器对象； BeanFactory和ApplicationContext在创建ioc容器的时候并获取对象的方式是有区别的，当通过前者来获取容器对象的时候只能加载和创建对象并不能初始化对象，而后者是只要加载到容器对象就会将容器里面的对象全部初始化；

ioc容器对象是如何实现管理对象的 通过反射，xml解析，工厂模式实现和加载容器中的对象管理 正常我们需要的是直接在service中new一个Dao对象，这样两个代码就耦合度太高接下来我们看看Spring是如何帮我们管理的； 通过工厂模式很友好的解决了耦合的问题，在工厂返回想要的对象之前，会进行xml解析，获取配置文件的位置并创建容器对象，通过反射的方式加载容器里面的对象。

Bean管理操作的两种方式

Spring中有两种Bean，一种是普通bean（定义的类型是什么返回的也是什么类型）第二种是工厂bean（FactoryBean）工厂bean的特点是定义的类型和返回的类型可以不一样；

  1. 普通bean
  2. 工厂bean-FactoryBean
  • 工厂bean和普通bean最大的不同就是在于定义的类型和返回的类型可以不同
  • spring提供了FactoryBean接口只需要实现该接口可以得到以下三个方法 Object getObject(); Class<?>getObjectType(); boolean isSingleton();
  • 主要是通过在接口方法（getObject();）中进行定义返回的对象的类型来决定返回类型；

bean的作用域

  • 在spring中我们可以设置创建的bean实例是单实例还是多实例；
  • spring默认创建bean是单实例的；
  • 如何设置单实例还是多实例
  • 通过属性scope来设置常见的值有两个分别是Singleton和prototype

Singleton的特点是默认设置的，就算不设置也是默认的； Singleton是在加载玩xml文件的时候就已经把bean单实例加载出来了， prototype是多实例的，如果设置成prototype在加载xml文件的时候并没有加载实例，而是在调用getBean()方法的时候加载多实例对象； 特点创建时机不同；

  • 基于xml配置文件方式实现 基于bean标签，标签里面添加对应的属性；

bean的生命周期

  • 从对象的创建到销毁的过程；
  • 创建bean实例（通过构造器创建）
  • 为bean的引用和是指属性（调用set方法）
  • 调用bean里面的初始化方法（需要配置）
  • bean可以获取到了可以 使用bean
  • 当容器在关闭的时候会调用bean的销毁方法（需配置）

以上就是基本的bean的生命周期

添加bean的后置处理器

  • 如果在bean的生命周期的基础上添加bean的后置处理
  1. 创建bean的实例对象（调用构造器）
  2. 通过对象的引用（内部应用和外部引用）（set方法进行属性赋值）
  3. 把bean的实例传递bean后置处理器的方法
  4. 调用bean的初始化方法（需要配置初始化方法）
  5. 把bean的实例传递给bean后置处理器的方法
  6. 使用bean实例（对象创建好了 可以使用）
  7. 当容器关闭后销毁并调用bean的销毁方法（需要配置销毁方法）
